# Personal Assistant MVP：后续推进介绍文档

## 1. 当前可展示能力（已完成）

当前系统已经能在本地完成以下闭环：

1. Telegram 消息同步入库（手动触发 + 定时轮询）。
2. AI/规则对消息做工作相关识别与任务抽取。
3. 联系人标签管理（manual / ai / pending）与 pending 一键确认。
4. Task 管理（创建、状态更新、assignee 指派、平台/分类筛选）。
5. Dashboard 展示（概览、联系人、任务、日历、设置）。
6. 日历视图融合（mock Google Calendar + Task 截止时间）。

## 2. 对外演示流程（建议 5-10 分钟）

### 2.1 演示目标

向观众清晰说明：系统已具备“聊天信息结构化 -> 可执行任务 -> 可视化管理”的最小可用产品能力。

### 2.2 演示前准备

1. 启动后端：`uvicorn app.main:app --reload --host 127.0.0.1 --port 8000`
2. 启动前端：`npm run dev -- --host 127.0.0.1 --port 5173`
3. 打开 Dashboard：`http://127.0.0.1:5173`
4. （可选）配置 Telegram Bot Token 后点击 `Sync Telegram`

### 2.3 演示脚本

1. 打开 `Overview`：
   说明平台消息分布、工作分类分布、今日任务与逾期任务统计。
2. 打开 `Contacts`：
   展示联系人标签，以及 pending 标签一键转 manual 的流程。
3. 打开 `Tasks`：
   展示自动抽取任务 + 手动新建任务；
   现场修改状态、assignee，并按平台/分类筛选。
4. 打开 `Calendar`：
   展示“日历事件 + 任务截止”统一视图。
5. 点击 `Generate Daily Summary`：
   说明系统可日汇总输出，便于个人复盘。

## 3. 后续实施路线图（建议 3 阶段）

## 3.1 阶段 A（1-2 周）：稳定化与可运营

目标：把当前 MVP 变成“可稳定日常使用”的版本。

1. 增加日志与可观测性：
   记录每次同步的成功/失败、重复消息数量、解析失败原因。
2. 数据质量治理：
   增加任务去重策略和标签冲突处理策略。
3. 基础测试完善：
   CI 执行后端测试、前端构建检查。
4. 本地部署脚本化：
   增加一键启动脚本（例如 `make dev`）。

验收标准：

1. 连续 3 天运行无阻断故障。
2. 消息重复入库率为 0（幂等有效）。
3. 关键 API 用例和核心页面构建在 CI 中通过。

## 3.2 阶段 B（2-4 周）：真实数据接入扩展

目标：从单渠道 Telegram 扩展到多渠道。

1. Gmail 接入（优先）：
   OAuth、邮件正文抽取、线程映射、任务抽取复用现有 AI pipeline。
2. 真实 Google Calendar 接入：
   用 OAuth API 替代当前 mock provider。
3. 平台统一抽象层：
   继续扩展 `platform_metadata`，统一 message/contact 映射协议。

验收标准：

1. Gmail 能稳定拉取近 7 天邮件并入库。
2. 真实日历事件可增量同步并正确展示。
3. 各平台消息可复用同一套任务抽取与标签逻辑。

## 3.3 阶段 C（4+ 周）：语音与智能助理升级

目标：实现你最初设想的“录音 + 多工具 + 实时助手”。

1. 语音输入（分两步）：
   第一步：支持音频文件上传转写；
   第二步：再做实时流式麦克风转写。
2. 助理增强能力：
   自动生成待办建议、跟进提醒、日程冲突提示。
3. 多人协作：
   Task assign 外发通知（Telegram 或 Email）。

验收标准：

1. 音频转写准确率达到预期阈值（按业务场景定义）。
2. 自动任务建议可被用户高比例采纳（可追踪采纳率）。
3. 分配通知链路具备可追踪投递结果。

## 4. 重点风险与合规注意事项

1. 聊天与语音数据涉及隐私，必须确保用户授权与最小化采集。
2. 接入第三方平台（Telegram/Gmail/微信）需符合各平台条款。
3. 录音功能在不同地区法律要求不同，默认应采取显式告知与同意机制。
4. 建议默认关闭原始内容长期存储，仅保留必要结构化信息。

## 5. 团队协作建议（最小角色）

1. Backend：平台接入、数据模型、API、任务抽取。
2. Frontend：Dashboard 交互、筛选、任务运营视图。
3. AI/Prompt：抽取准确率、分类标准、误报控制。
4. PM/运营：标签体系、任务流程、验收标准定义。
